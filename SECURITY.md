# 安全政策

## 支持范围

当前仅维护 `main` 分支的最新版本。Abar 仍处于早期阶段，不承诺旧版本的长期安全更新。

## 报告安全问题

请不要公开创建包含漏洞细节、令牌、认证文件或个人数据的 Issue。

优先使用 GitHub 仓库的 **Security → Report a vulnerability** 私下报告。如果该入口不可用，请联系仓库维护者，并仅提供复现所需的最少信息。

报告建议包含：

- 受影响版本或 commit
- 问题影响
- 最小复现步骤
- 建议修复方式（如果有）

请勿附加真实的 `auth.json`、访问令牌、Cookie 或未经清理的数据库。

## 安全边界

- 本地事件服务应只监听 `127.0.0.1`。
- Reporter 不应阻塞 Codex，也不应向远程主机发送 Hook 数据。
- Abar 会读取本地 Codex 认证状态以查询额度，但不得持久化认证凭据。
- 当前构建采用 ad-hoc 签名，未经过 Apple 公证。
